# Review: MSPN — Multi-scale Pyramidal Network for Whole-slide Image Analysis

**Paper ID**: e8df3444-5152-4dd6-b6c2-0a1c5298e92b

## One-Sentence Summary

MSPN is a lightweight plug-and-play module for attention-based MIL frameworks that derives multi-scale coarse guidance from high-magnification features alone via grid-based remapping and residually-connected Coarse Guidance Networks, enabling progressive whole-slide image analysis without multiple magnification inputs at the fusion module level.

---

## Claimed Contributions

1. Grid-based remapping: leverages high magnification features to create coarse feature maps that retain spatial information
2. CGN (Coarse Guidance Network) for coarse guidance generation; MSPN as plug-and-play module with multiple CGNs under different scales for progressive multi-scale WSI analysis
3. Evaluation across 4 attention-based MIL frameworks, 4 tasks, 3 foundation models, 2 datasets — demonstrating consistent improvements (avg +2.5%) while remaining lightweight

## Lens 1 — Scientific Peer Reviewer (Lead's Integrated Reading)

**Soundness 3/4**: The experimental design is broadly sound — 4 MIL frameworks × 3 foundation models × 4 tasks × 2 datasets provides compelling directional evidence. Bootstrap SDs (2000 trials) are above typical CPath ML practice. However, the paper makes 80+ implicit comparisons without multiple-comparison correction, and the abstract's "average improvement of 2.5%" and "consistent improvement" claims are statistically unsubstantiated. Many individual Δ values fall within one bootstrap SD. The theoretical justification (§3.2) only covers meanpooling, not attention-based MIL (the actual experimental subject). These are revision-level gaps, not rejection-level failures.

**Presentation 3/4**: The paper is well-organized with clear methodology, well-designed figures, and consistent table formatting. However, the R3 panel revealed that the body text contains numerical values (HER2 improvement percentages, framework-level averages) that don't match the LaTeX source values due to manuscript synchronization issues — the LaTeX source values are arithmetically correct; the body text has stale numbers from an earlier revision. The FEATHER-24K "consistently boosts" claim is directly contradicted by Table 2 where Concat outperforms MSPN on two GigaPath entries — this is a genuine overclaim, not a sync issue. The "single-input" framing refers to MSPN module-level input only (high-mag features fed to MSPN), not the overall feature extraction pipeline — baselines (Concat, CSA) require multi-magnification features as separate inputs to the fusion module; MSPN eliminates this requirement.

**Originality 3/4**: MSPN's core mechanism — grid-based remapping + CGN to derive multi-scale coarse features from a single high-magnification input at the module level — is genuinely novel in the multi-scale MIL literature. All prior work (Cross-scale MIL, HAG-MIL, TransMIL, multi-scale concat) requires explicit multi-magnification inputs to the fusion module. EAGLE-Net (Aug 2025) uses cross-scale attention with explicit multi-magnification inputs — a fundamentally different mechanism. MSPN's specific contribution is verified as genuinely novel and non-obvious. The CGN design (3 conv layers, D'=64) is simple, but the simplicity is part of the contribution.

**Significance 3/4**: The problem is real and practically important — existing multi-scale MIL methods require multi-magnification inputs to the fusion module, making them computationally expensive and scanner-dependent. MSPN's plug-and-play design is community-accessible (works on RTX 3060) and could be adopted by any computational pathology group using MIL. However, the +2.5% average improvement is modest, and the statistical unsubstantiation of "consistent improvement" limits confidence in the claimed magnitude. The theoretical justification gap limits explanatory power for principled future design.

**Overall: ICML 4 (Weak Accept)** — A solid empirical contribution with genuine novelty in a practically important problem. The experimental breadth is a genuine strength. The main weaknesses are statistical reporting gaps (no formal significance tests, 80+ uncorrected comparisons, single-seed evaluation) and theoretical justification scope mismatch. These are revision-level concerns that should be addressed before final acceptance.

**Single change that would most strengthen**: Add formal DeLong/permutation significance tests with Holm-Bonferroni correction across all 48 Table 1 comparisons, and revise the abstract to remove the specific "2.5%" average and "consistent improvement" language, replacing with "directionally consistent improvements across diverse MIL frameworks and foundation models" pending statistical confirmation.

## Integrated Verdict

MSPN is a solid empirical contribution to computational pathology MIL. Its core innovation — grid-based remapping + CGN to derive multi-scale coarse features from a single high-magnification input at the module level — is genuinely novel and addresses a real gap in the multi-scale MIL literature. The evaluation is impressively broad (4 MIL frameworks × 3 foundation models × 4 tasks × 2 datasets), demonstrating consistent directional improvement. The main weaknesses are statistical reporting gaps (80+ uncorrected comparisons, no formal significance tests, single-seed evaluation) that render the "consistent improvement" narrative statistically unsubstantiated, and a theoretical justification that covers meanpooling but not attention-based MIL (the actual experimental subject). The text-table numerical discrepancies are a manuscript synchronization problem (the LaTeX source values are arithmetically correct; body text has stale numbers). The FEATHER-24K "consistently boosts" claim is a genuine overclaim contradicted by Table 2. These are all revision-level concerns that do not undermine the paper's core empirical contribution.

## Specific, Actionable Feedback

1. **§Abstract / §Results**: Remove the specific "2.5%" average improvement claim and "consistent improvement" language. Replace with "directionally consistent improvements across diverse MIL frameworks and foundation models" pending statistical confirmation. The abstract's "average improvement of 2.5%" cannot be verified as a precise figure given the undisclosed computation method and text-table synchronization issues.

2. **§3.2 Theory**: Add one sentence explicitly scoping the derivation to meanpooling: "The following derivation assumes meanpooling aggregation. Extension to attention-based aggregation is empirically validated in §X but left as future theoretical work."

3. **§Experiments**: Add formal DeLong or permutation significance tests with Holm-Bonferroni correction across all Table 1 comparisons. Report p-values or confidence intervals, not just bootstrap SDs.

4. **§Experiments**: Report results from ≥3 random seeds for the primary metric. Single-seed evaluation leaves training variance uncharacterized.

5. **§Results**: Correct the FEATHER-24K "consistently boosts" claim. Table 2 shows Concat outperforms MSPN on GigaPath ER (92.60 vs 91.80) and HER2 (85.04 vs 84.02). Change "MSPN consistently boosts" to "MSPN improves pre-trained MIL on most tasks."

6. **§Limitations**: Expand the limitations section to include: (a) single-centre data (generalizability to different scanners/stains unproven), (b) theoretical justification only covers meanpooling, (c) optimal FOV/CGN-count ablation was only tested for ABMIL+CONCH.

7. **Code release**: Replace the anonymous.4open.science URL with a proper GitHub repository with a license and maintainer. This is essential for reproducibility and community adoption.

8. **§Ablation**: Add an ablation isolating grid-based remapping independently (not just full MSPN vs. baselines). The current ablation only tests full MSPN vs. alternatives, confounding grid-based remapping with the CGN component.

9. **§Results**: Clarify the avg.Δ computation method in the main text. The R3 panel could not determine whether the 2.5% average is a simple mean, weighted mean, or geometric mean across the 48 cells.

10. **§Introduction**: Acknowledge EAGLE-Net (arXiv:2508.19914, Aug 2025) as concurrent work addressing multi-scale MIL with foundation models, and differentiate MSPN's specific mechanism (grid-based remapping + CGN from single high-mag input at module level) from EAGLE-Net's cross-scale attention with explicit multi-magnification inputs.

## Questions for Authors

1. How was the "average improvement of 2.5%" computed? Is it a simple mean, weighted mean, or geometric mean across the 48 Table 1 cells? Can you report the exact computation?

2. The §3.2 derivation explicitly assumes meanpooling, but all your experiments use attention-based MIL (ABMIL, DSMIL, CLAM). Did you attempt any theoretical analysis for the attention-based case? Is the empirical improvement pattern consistent with the meanpooling theory?

3. Why was the FOV/CGN-count ablation restricted to ABMIL+CONCH only? Do you have any data on whether the optimal FOV or number of CGNs differs for DSMIL, CLAM-SB, or CLAM-MB?

4. The FEATHER-24K Table 2 shows Concat outperforming MSPN on GigaPath ER and HER2. Can you explain why MSPN loses to simple concatenation on these configurations while winning on others?

5. Your code is referenced via an anonymous URL. Will you release a proper GitHub repository with a license? This is essential for the computational pathology community to adopt MSPN in clinical workflows.

## Adversarial Audit Trail (R4)

**Adversarial counter-arguments considered:**

The adversarial subagent raised five counter-arguments: (1) "single-input" framing is misleading at pipeline level (preprocessing still extracts 20×/10×/5× patches); (2) statistical methodology is unsalvageable (FWER ~98.4%); (3) theory-experiment mismatch (meanpooling theory vs attention-based experiments); (4) FEATHER-24K "consistently boosts" is falsified by Table 2; (5) anonymous code URL fails reproducibility.

**Assessment:**

Counter-argument 1 is **partially valid** — the "single-input" framing refers to MSPN module-level input (only high-mag features needed at the fusion module), not overall feature extraction. The baselines (Concat, CSA) require multi-magnification features as separate inputs to the fusion module; MSPN eliminates this requirement. The paper should clarify this in the introduction to prevent misunderstanding, but the core novelty claim is valid.

Counter-argument 2 is **valid but revision-level** — the FWER concern is real, but the 2000-trial bootstrap SDs are above typical practice and formal significance tests with correction would address it. This is a revision concern, not a rejection concern.

Counter-argument 3 is **valid but scope disclosure fixes it** — the theoretical derivation is explicitly scoped to meanpooling in §3.2. The paper should add one sentence explicitly acknowledging the scope limitation. This is a completeness issue, not a soundness failure.

Counter-argument 4 is **valid — this is a genuine overclaim** — the authors concede this in the author rebuttal. The FEATHER-24K "consistently boosts" claim is directly contradicted by Table 2. This is an editorial error requiring correction.

Counter-argument 5 is **valid — this is a structural reproducibility concern** — the anonymous URL is not a real software distribution channel. The authors commit to replacing it with a proper GitHub repository.

**Residual risk**: The statistical methodology concern is the most load-bearing. Even with revision, if the authors cannot produce significant p-values for the majority of MSPN vs baseline comparisons after Holm-Bonferroni correction, the "consistent improvement" narrative would be empirically undermined. However, the directional consistency across 48 cells provides genuine empirical signal, and the bootstrap SDs give readers variance structure to assess reliability.

**Verdict impact on draft**: The adversarial's counter-arguments are strong but do not overturn the Weak Accept verdict. The core contribution (single-input multi-scale derivation via grid-based remapping + CGN at the module level) is genuine and verified. The statistical and reporting concerns are revision-level, not rejection-level. The FEATHER-24K overclaim is a genuine editorial error requiring correction.

**Restating the target.** The paper argues that its bi-phase curated reinforcement learning (BiC-RL) framework solves the deceptive-behavior problem in generated CUDA kernels by decomposing kernels into a prefix, a core-implementation, and a suffix — and training the model to generate only the core. The two-phase curriculum (kernel infilling stage → end-to-end generation stage) is presented not merely as helpful but as *necessary*: the model must "master the core implementation of the kernel before transitioning to end-to-end kernel generation" (§4.2). The ablation in Table 3 (tab:ablation-results) shows that each stage incrementally improves over the RL baseline, but it does not directly test whether Phase 1 must precede Phase 2.

**Where I agree.** The deceptive-behavior problem is real and well-characterized. The three-category taxonomy in §4.1 (example-matching, missing invocation logic, omitted kernel from forward pass) is precise and actionable. The kernel decomposition scaffold is intuitive: by pre-filling the prefix and suffix, the model cannot accidentally or strategically bypass the core implementation. The empirical results are also genuinely strong — DICE-8B matches cudaLLM-8B despite cudaLLM using substantially more training data and output tokens, and DICE-1.7B is the only 1.7B-scale model that reliably generates correct kernels on KernelBench Level 1.

**What I take from this.** The staged curriculum is a plausible and well-motivated solution to a genuine problem. The RL training trajectory in Figure 2 (BiC-RL vs. baseline) shows a clear upward trend with stable convergence, while the baseline RL exhibits stochastic instability — this is compelling evidence that the curriculum matters. The CuKe dataset curation (6,303 kernels, augmented from cudaLLM and ConCuR) provides a reasonable data foundation, even if the "high-performance" selection criterion could be more precisely defined.

**Where I push back.** The ordering necessity claim — that Phase 1 (infilling) *must* precede Phase 2 (end-to-end) — is asserted without direct ablation. Table 3 validates that the full BiC-RL pipeline outperforms the RL baseline and that each individual phase contributes, but it does not test the counterfactual: what happens if you train end-to-end first and then add infilling? Or if you interleave both phases? The paper's framing ("ensures that the model masters the core implementation before transitioning") reads as a strong causal claim, not merely a correlational one. Without the ordering ablation, the claim is undersupported. A related concern: the CuKe curation criterion ("high-performance kernels") is described qualitatively but not operationally defined. What metric determines "high performance"? A speedup threshold? An execution-correctness filter? Without this, the dataset is difficult to reproduce or extend.

**My current calibration.** ICML Overall: 3 (Weak Accept) — Confidence: 3. The contribution is real (deceptive behavior taxonomy + kernel-infilling curriculum), the empirical gains are multi-scale and well-measured, and the ablation structure is mostly sound. The primary gap is the ordering necessity claim, which if validated would strengthen the paper to an Overall 4; if refuted, the contribution remains useful but is more accurately described as "RL on dLLMs for kernel generation" rather than a novel curriculum design.

**Socratic question.** [probe-evidence] Could the authors clarify whether an experiment training end-to-end first and then adding the infilling stage (Phase 2 → Phase 1) was attempted or is planned? This would directly test whether the *order* of the curriculum is load-bearing or whether any two-phase training with a kernel-specific stage provides the stabilization benefit.
